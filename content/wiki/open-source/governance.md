## Roles

| Role | Responsibility |
| ---- | ---------------- |
| **Maintainer** | Triage, review, release, roadmap input |
| **Committer** | Merge rights on a repo |
| **Contributor** | Anyone with merged PRs |

## Decision process

1. Discuss in issue or RFC for large changes
2. Maintainer proposes; lazy consensus after 5 business days
3. Deadlock → escalate to engineering lead

## Adding maintainers

- Consistent quality reviews over ~3 months
- Nominated by existing maintainer; announced in repo README

## Archiving repos

Archive when unused 12+ months; document redirect or successor in README.
