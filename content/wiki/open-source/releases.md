## Versioning

- Follow **SemVer** for libraries and apps
- Tag releases on GitHub with changelog notes

## Release checklist

- [ ] CI green on release branch
- [ ] Changelog updated (`## Unreleased` → version)
- [ ] Security scan / dependency audit for critical CVEs
- [ ] Contributor credits in release notes
- [ ] Announce in `#announcements` if user-facing

## Security fixes

- Embargo discussion in `#security` (private channel)
- Patch release ASAP; credit reporters if they agree

See [Volunteer & Contributor](/wiki/volunteer-contributor/overview) for external PR flow.
