## PRD outline

1. Summary & goals (measurable)
2. User stories / jobs to be done
3. Requirements (must / should / could)
4. UX notes + Figma links
5. Analytics events
6. Rollout & feature flags
7. Open questions

## Acceptance criteria

Use **Given / When / Then** for anything testable:

> Given a new member joins an event, when they complete registration, then they receive a confirmation email within 5 minutes.

## Review checklist

- [ ] Engineering reviewed feasibility
- [ ] Design linked mocks
- [ ] Privacy / legal flagged if PII involved
- [ ] Success metrics defined before build starts
